# 个人网站需求计划书（Portfolio Web Canvas）

## 1. 项目概览

**网站目的**

- 展示我的个人作品与研究项目（portfolio / lab）。
- 用一种「桌面 / 操作系统」式的 canvas 交互来体现个性与设计感。
- 未来可以作为求职、申请学校时的作品入口。

**整体风格关键词**

- 现代、简洁、略带“系统 UI / OS 桌面”感。
- 暗色系 canvas（深灰）+ 深蓝色弹窗。
- 字体：Helvetica（web 上用 `font-family: "Helvetica", "Arial", sans-serif;` 做 fallback）。

---

## 2. 信息架构（IA）

### 2.1 顶层结构（页面）

1. **Loading 页面**（进入网站时短暂展示）  
2. **主页：作品 canvas**
   - 中间一块大 canvas（深灰色）区域。
   - 多个可拖拽的小弹窗（每个代表一个作品或一类内容）。
   - 一个略大一些的弹窗：个人资料 / Profile。
3. （可选）**作品详情页**  
   - 点击弹窗中的“More / View case study”后跳转到单独的作品详情页面（或在同一页打开全屏 overlay）。
4. （可选）**联系 / Contact 区块**
   - 可以是 canvas 底部的固定区域，或某个弹窗展开后展示。

---

## 3. 视觉与动效需求

### 3.1 Loading 动画

- **背景**：纯白 `#FFFFFF` 全屏。
- **中间 UI 元素**：
  - 一个简洁的 loading 标识（例如圆圈旋转，或者一个小方块在左右移动）。
  - 下面有一行简短文字，例如：`Loading Roma’s workspace…`
- **动效要求**：
  - 进入时淡入加载 UI。
  - 页面资源加载完成后，白色背景淡出（fade out），灰色 canvas 从中间轻微缩放+淡入出现。
- **时长**：如果资源加载很快，也至少展示 0.8–1 秒，避免闪一下的感觉。

### 3.2 Canvas（主页背景）

- **颜色**：深灰 `#1E1E1E` 或类似。
- 占满视窗（full viewport），支持响应式。
- 视觉上类似一个“工作桌面”：  
  - 可以有非常轻微的渐变或噪点纹理（不要太花）。

### 3.3 弹窗（窗口）

- **基础样式**：
  - 背景颜色：深蓝 `#082B5F`（可略微调亮，保证可读性）。
  - 圆角：8–12px。
  - 有轻微阴影，营造悬浮感。
  - 顶部有一个“标题栏”（显示作品名或“Profile”等）。
  - 右上角可以做最小化 / 关闭图标，但不必完全复制操作系统。
- **字体**：
  - 全站：`font-family: "Helvetica", "Arial", sans-serif;`
  - 标题：稍大的粗体；正文：常规权重。
- **状态变化**：
  - 默认状态：小尺寸窗口。
  - 被点击（获得焦点）：
    - 放大一点（例如缩放到 1.05 或增加宽高）。
    - 阴影增强。
    - 提升 z-index，覆盖其他窗口。
- **Profile 弹窗特别要求**：
  - 比其他窗口略大（例如默认宽 520–640px）。
  - 展开时可以从“概览模式”变成“详情模式”，显示更多内容（教育背景、经历、下载简历按钮等）。

## 4. 交互设计需求

### 4.1 拖拽弹窗

- **交互方式**：
  - 用户可用鼠标抓住弹窗标题栏，拖拽到 canvas 任意位置。
- **行为约束**：
  - 窗口不会被拖出视口（或至少标题栏不会超出顶部和左右边界）。
- **状态**：
  - 正在拖拽时：略微降低透明度或加描边，提示已被选中。
  - 松开时：轻微的“回弹”动效，让体验更顺滑。

### 4.2 弹窗点击行为

- **单击**：
  - 设为当前激活窗口（z-index 最高），略微放大。
- **双击或点击窗口中的「Expand / Open」按钮**：
  - 进入该作品的“放大视图”：
    - 方案 A：弹窗居中放大，占据 60–80% 宽度，高度自适应。
    - 方案 B：打开一个全屏 overlay（作品详情），返回时回到 canvas。

### 4.3 Profile 弹窗

- **默认显示信息**：
  - 名字（大字号）。
  - 短句式自我介绍（1–2 句）。
  - 一两个关键词标签（Architecture / Computational Design / etc.）。
- **展开后显示**：
  - 教育经历（学校、学位、时间）。
  - 工作 / 实习经历精选。
  - 技能列表（短标签）。
  - 联系方式（邮箱、LinkedIn、CV Download 按钮等）。

### 4.4 响应式设计

- **桌面端（主体验）**：
  - 完整的拖拽窗口体验。
- **平板 / 手机端（简化方案）**：
  - 拖拽体验可以弱化。
  - 方案：将弹窗变为可滑动的卡片列表（上下滚动），点击卡片进入详细视图。
  - 背景仍可保持深灰，但重心从“桌面”转为“作品列表+卡片”。

## 5. 内容与模块规划

### 5.1 作品弹窗内容结构（每个项目）

每个作品弹窗内：

1. 标题：项目名称。
2. 简短一句话概述（1 行）。
3. 1 张缩略图 / 代表性图像。
4. 标签：如 “Computational Design / Urban / Installation / etc.”
5. 按钮：`View more` or `Case study` → 打开详细视图。

### 5.2 作品详情视图（可选）

包含：

- 项目简介（背景 / 目标）。
- 个人贡献说明。
- 关键图像（可左右滑动或图片画廊）。
- 关键技术（软件 / 技术栈）。
- 结果与反思（简短文本）。

### 5.3 个人资料内容（Profile）

- 简要自我介绍文案。
- 教育背景。
- 代表性经历（公司 / 工作室 / 研究实验室等）。
- 技术栈 & 兴趣方向（标签式）。
- 联系方式 & CV 下载链接。

## 6. 技术实现与 AI 协作方式（建议）

> 这一部分是给之后用 AI 写代码时的「技术路线建议」，也可以按需调整。

### 6.1 前端技术栈（建议）

- **框架**：Next.js（React）或纯 React SPA。
- **语言**：TypeScript（可选，但推荐）。
- **样式**：
  - CSS-in-JS（如 styled-components）或 Tailwind CSS（二选一即可）。
- **拖拽实现**：
  - 使用现成库（例如 `react-draggable`）来实现弹窗拖拽。
- **字体**：
  - 在全局样式中定义 `font-family: "Helvetica", "Arial", sans-serif;`。

### 6.2 AI 使用场景

1. **结构搭建**：  
   - 用 AI 帮你生成 Next.js 项目基础结构（pages、components）。
2. **组件级实现**：
   - 让 AI 写：`Canvas` 组件、`Window`（弹窗）组件的 React 代码。
3. **动效与细节调优**：
   - 提示 AI：增加拖拽状态样式、点击放大动画、Loading 动画。
4. **多端适配**：
   - 让 AI 根据你给的设计说明，生成 mobile breakpoint 的 CSS。
5. **文案润色**：
   - 用 AI 帮你写 / 改作品介绍文案中英文版本。

## 7. 上线与域名规划

### 7.1 部署方式（推荐）

- 代码托管：GitHub / GitLab。
- 部署平台（可选其一）：
  - Vercel（与 Next.js 非常匹配）。
  - Netlify。
- 要求：
  - 自动部署：每次 push 到 main 分支即可自动更新网站。
  - HTTPS（免费证书，一般平台会自动配置）。

### 7.2 域名规划

- 选择一个简单好记的域名，例如：  
  - `romaluo.com` / `roma-portfolio.com` 等。
- 域名注册平台：  
  - Namecheap、Google Domains（视地区而定）、阿里云等。
- 域名解析设置：
  - 将域名的 DNS 解析指向部署平台（例如 Vercel 提供的 Nameserver / CNAME）。
- 要求：
  - 支持 `https://` 访问。
  - 设置 `www` 与根域名互相跳转（如 `www.roma.com` → `roma.com`）。

---

## 8. 性能与基础配置

- **性能**：
  - 图片尽量压缩（WebP / AVIF）。
  - 使用懒加载加载非首屏图片。
- **SEO 基础**：
  - 每页设置 `<title>`、`<meta description>`。
  - Open Graph（OG）标签，便于分享时展示封面图。
- **统计**（可选）：
  - 接入 Google Analytics 或 Plausible 收集访问数据。

---

## 9. 项目分阶段计划

1. **第 1 阶段：结构搭建**
   - 定好技术栈（React / Next.js）。
   - 生成基础项目 + 路由结构（Loading 页 + canvas 主页）。
2. **第 2 阶段：UI 组件**
   - 实现深灰 canvas。
   - 实现可拖拽弹窗组件（最小 demo）。
3. **第 3 阶段：交互与动效**
   - 实现点击放大、聚焦、z-index 管理。
   - 实现 Profile 弹窗的展开 / 收起。
   - 完成 Loading 动画 → 主页过渡。
4. **第 4 阶段：填充内容**
   - 整理作品列表（图与文案）。
   - 将作品内容填入各个弹窗与详情页。
5. **第 5 阶段：适配与优化**
   - 移动端布局适配。
   - 性能、加载速度优化。
6. **第 6 阶段：上线**
   - GitHub 仓库配置。
   - 连接 Vercel / Netlify 部署。
   - 绑定域名，测试正式访问。
